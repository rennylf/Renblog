想法背景
----

在2019 年，我就开始了写日记。它不仅能缓解我的存在主义危机，也能给生活带来一点掌控感。为了更方便写和存，我所写的是数字日记，而不是物理日记。然而，即使是数据日记，我还是没有 5 年如一日地写，甚至多次中断，大概自己是有瘾却懒惰的人。因此，我常想如何才能毫不费力保持日记写作。在这期间，我换过很多工具，从桌面端的应用到手机端都，始终没有找到那一个，现在，我终于有了个相对好用的方案（我希望是 End game）。

现有痛点
----

现有写数字日记的工具大致可以分为两类：笔记类应用和日记类应用。

首先是笔记类应用，比如OneNote，Obsidian，Apple Notes等，它们并非围绕日记场景设计，因此总是需要用户自行去设定笔记本/文件夹，当你想写时，就需要找到设定好的位置，手动建立日记，输入日期。每每想这一通操作，拖延症就会发作，觉得写日记是个大麻烦。

另一方面，日记类应用，例如Day one，似乎能够免去一些手工。它们的界面更加精美，往往具有针对日记的功能，例如“获取当日的天气”。但这类应用对我来说，有个命定的缺点，它们很封闭，不提供日记批量导出，因此，数据不能够掌控在自己手中，这会让我没有安全感。

需求分析
----

仔细想想，我理想中的日记系统应该满足如下需求：

1.  光速启动：作为懒人，萌生写作想法的瞬间，我希望马上开始写，尽可能缩短进入写作状态的路径。
    
2.  管理自动化：作为强迫症患者，我时常想按照一定的秩序管理所有日记，比如统一它们的命名方法，内容格式；按年月的维度把它们分文别类。人工做这些，太消耗时间和精力。
    
3.  本地优先：Local first 一直是我追求的理念，我认为最带给人安全感的应用应该是采用开放的文件格式，数据完全保存在本地，能够给用户最大的自由度。
    

构建思路
----

同时实现这三个需求并不容易，至少我目前没有找到这样的应用。然而，受到 Chatgpt 编程的启发，我认为大概可以通过多个工具的配合，在桌面端实现这三个需求，它们的核心是用 Markdown 格式来写作日记，并搭配如下工具：

1.  启动器：可用系统自带或者第三方的启动器，作用是能够快速检索并打开日记文件，
    
2.  管理秘书：由 Chatgpt 来承担，作用是根据我的管理诉求，写出可用的 python 脚本（需搭建 python 运行环境），帮助我实现自动化管理，例如创建本年年度的所有日记文本。
    
3.  编辑器：选用一款轻量的桌面端 Markdown 编辑器（移动端同样存在编辑器，可以作为辅助），作用是能够作为默认的 Markdown 文件打开和编写工具。
    
4.  云服务：选用一款云服务，作用是自动备份和同步日记文件，避免数据丢失，同时实现多个设备浏览和编辑。
    

大致的使用思路是：① 先明确日记的格式以及组织方式，将生成日记需求告诉 Chatgpt ，让它写作脚本。② 然后，我们在本地运行脚本（云服务的文件夹），自动生成所有日记文件。③ 每次我们想要写日记的时候，可以用使用启动器来检索并打开日记，使用默认编辑器来写作。④ 最后使用云服务来备份和同步日记。

系统实例
----

下面提供在 MacOS 中的一个实例，它由 Spotlight + Chatgpt & Python + Typora + Onedrive 实现。总结来说，就是两个主要场景：A 日记生成（一次性的）；B 日记写作（周期性的）

日记生成效果

我会根据自己的需求写作提示词，让 Chatgpt 生成符合自己需求的脚本（例如设定每篇日记的模版等），并在Onedrive 中文件中执行。下文的提示词仅用于生成空白文档，可供参考。为了执行 Python 脚本，需要先下载安装 python，然后打开python的IDLE，将脚本代码输入后保存，点击菜单栏中的” run” 即可。详细可以参考其他教程。

> “
> 
> 请帮助我写一个 python 脚本，用来完成如下任务。在当前文件夹中，创建 12 个子文件夹。分别以 2024 + 月份命名。例如202401；在每个子文件夹中，创建一系列 Markdown 文档，与子文件夹的月份对应，生成当月所有日期的文件。使用2024+月份+日命名，例如20240101。注意：Markdown 文件不要添加任何内容。
> 
> ”

日记写作效果

① 一般来说，我会在系统的提醒事项中设立一个循环的提醒，在晚上 10 点提醒我写日记，作为一个很有效的推动手段。然后，我会使用 Spotlight 搜索日记，并用 Typora 作为默认编辑器打开和写作，Typora 无需手动保存，关闭默认保存，Onedrive 会在后台帮助我把笔记备份和同步在云端。

总结与思考
-----

本文基于我个人的需求，构思了一个能够光速打开、自动化管理和本地优先的日记系统，并且提供了一个在 MacOS 上的实例。这个系统能够有效缩短从产生写作想法到投入写作的路径，减少因为嫌麻烦所导致的拖延行为。同时，它借助Chatgpt 和 python 脚本，高效地满足强迫症患者的管理欲望，此外，它还将数据的掌控权完全还给用户。

在实践这套日记体系几天后，我有了一些新的感悟。

首先，格子法有助于持续写作。格子法是《提问写作法》中介绍过的一个方法，将一页纸划分出9 个格子，在格子中写上关于你创作的问题，然后你只需要回答这些问题，填满这些格子，就能够完成你的创作。它解释这个方法的原理就是人类填满格子的欲望。很多产品我们都可以看到这个思路，如 Github、语雀和 flomo的热力图，它们都用颜色填充格子的方式，可视化用户每日的活跃程度。我想，事先规划好一整年的日记，也相当于给自己创建了很多格子。接下来的一年，我不过是要填满这些格子。在填格子过程中，我确实收获了很多成就感，并且有持续下去的动力。格子法的思维可以拓展到创作场景，事先梳理出所有的工作，做好准备，一是避免准备工作和创作工作混在一起，增加创作的困难，二是提供一个全貌，让产生更多完成的动力。

其次，使用Chatgpt写脚本提升效率是可行的。回看过去铺天盖地的 python 培训广告，宣传学习 python 提升效率，但普通人即便是花了时间和金钱，大部分都是从入门到入土。而现在，在 Chatgpt 的帮助下，我们很轻松能够得到自己想要的代码，去处理一些重复性的工作，而我们只需要关注自身的需求，在本文中生成日记文件是一个典型例子，其他的还有很多，以后有机会会再讨论。然而，这并不是说完全不用学 python ，一些基础是必要的，例如 python 的环境搭建、基本语法、库的相关知识等，这些知识在免费的在线学习平台就可以获取（强烈推荐北京理工大学嵩天老师的系列课程），了解基础就能帮助你真正的把 Chatgpt 和python 的组合用起来。
